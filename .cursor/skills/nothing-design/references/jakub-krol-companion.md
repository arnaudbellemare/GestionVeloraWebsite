# Jakub Król — companion craft (Nothing-aligned)

Optional refinement layer for **motion, drag, modal composition, and design-engineer workflow**. Nothing’s typography-first, low-chrome rules in the main skill still win unless the user explicitly overrides them.

## Further reading

- [Motion gestures](https://jakub.kr/work/motion-gestures) — tie motion to hierarchy; avoid decorative animation.
- [Drag gesture](https://jakub.kr/work/drag-gesture) — pointer patterns; keep feel **percussive / precise**, not elastic.
- [Using AI as a design engineer](https://jakub.kr/work/using-ai-as-a-design-engineer) — agent-friendly spec and build workflow.
- [Sign-in dialog](https://jakub.kr/components/sign-in-dialog) — focused overlay composition and states.
- [Gradients](https://jakub.kr/work/gradients) — CSS/OKLCH craft. **Nothing forbids UI chrome gradients by default.** When the user wants gradients, use the **nothing-gradients** skill if it exists in the project (e.g. `.cursor/skills/nothing-gradients/SKILL.md`).

## Conflict rule

Nothing anti-patterns (no bounce easing, blur chrome, gradient stacking on every card, etc.) apply unless the user explicitly relaxes them. Use this material for **timing, gesture clarity, and structure** — not as a license to break the baseline system.
